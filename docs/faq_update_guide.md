# FAQ データ更新ガイド

## 概要

Q&Aボットの回答は `faq_data.txt` の内容に基づいています。
FAQ を追加・修正すると、次回のアプリ再起動時に自動で反映されます。

## ファイルの場所

```
faq_data.txt   ← これを編集する
```

## 書き方のルール

### 基本フォーマット

```
Q: 質問文をここに書く
A: 回答文をここに書く。複数行にわたっても OK。
```

- **Q:** と **A:** のペアで1つの FAQ になります。
- Q と A の間に空行は入れないでください。
- FAQ 同士は **空行1行** で区切ってください。

### セクション見出し

`#` や `##` で見出しを付けると、検索精度が上がります。

```
## イベント情報

Q: 次の新歓はいつですか？
A: 4/17（金）19:00〜 渋谷で開催します。

## 入会方法

Q: 入会するにはどうすればいいですか？
A: LINE公式アカウントから「入会希望」とお送りください。
```

### 自由記述セクション

Q&A 形式でなくても、箇条書きや説明文はそのまま書いて大丈夫です。
ボットが関連する部分を検索して回答に使います。

```
## Hello AI 基本情報
* 設立: 2025年5月
* スポンサー: 東京大学松尾研発スタートアップ 2WINS
```

## 更新手順

### 1. ローカルで編集

1. `faq_data.txt` をテキストエディタで開く
2. FAQ を追加・修正する
3. 保存する（文字コードは **UTF-8**）

### 2. ローカルで動作確認

```bash
streamlit run app.py --server.port 8504 --server.fileWatcherType none
```

追加した質問をボットに投げて、正しく回答されるか確認する。

### 3. デプロイ

```bash
git add faq_data.txt
git commit -m "FAQ を更新"
git push
```

GitHub に push すると、Streamlit Community Cloud が自動で再デプロイします。

## よくある更新パターン

### FAQ を追加する

ファイル末尾、または該当するセクションの末尾に Q&A を追記します。

### イベント情報を差し替える

`## イベント情報` セクションの中身を新しい情報に書き換えます。
終了したイベントは削除するか、「過去のイベント」セクションに移動してください。

### SNS の URL を変更する

`faq_data.txt` 内の URL を修正し、`config.py` のデフォルト値も合わせて更新してください。

## 注意事項

- ファイルの文字コードは必ず **UTF-8** にしてください（メモ帳で保存する場合は「UTF-8」を選択）。
- 1つのチャンクが長すぎると検索精度が落ちます。1つの Q&A は **500文字以内** を目安にしてください。
- 変更後にローカルで確認してから push するのがおすすめです。

## Supabase ログの活用

`question_logs` テーブルの `is_in_faq = false` の行は「FAQ にない質問」です。
定期的に確認して、よくある質問を `faq_data.txt` に追加してください。
